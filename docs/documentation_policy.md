# Documentation Policy

## Principles
1. Documentation is part of the deliverable.
2. `README.md` is the primary entrypoint.
3. `/docs` contains detailed procedures, architecture notes, and operational references.

## Update Rules
1. Update `README.md` when setup, run commands, or high-level features change.
2. Update `/docs` when architecture, workflows, integrations, or behavior changes.
3. Keep examples accurate and executable.
4. Do not document unimplemented behavior as complete.

## Required Docs For Major Features
1. Feature behavior and user flow.
2. Configuration and environment variables.
3. Testing coverage notes.
4. Known limitations and tradeoffs.
